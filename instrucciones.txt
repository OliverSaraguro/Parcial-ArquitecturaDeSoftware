MEGAPROMPT: DESARROLLO DE SISTEMA SGV-APS (VALIDACIÓN DEEPWIKI)
================================================================================

ROL: Actúa como un Arquitecto de Software Senior con especialidad en Clean Architecture.

OBJETIVO: 
Desarrollar el "Sistema de Gestión y Venta de Artesanías del Pueblo Saraguro (SGV-APS)" 
bajo una arquitectura estrictamente separada en 3 capas en un único nodo, para superar 
la auditoría técnica de DeepWiki (puntuación máxima 3/3 en arquitectura).

DURACIÓN DEL DESARROLLO: 120 minutos (Optimiza el código para ser funcional y elegante).

--------------------------------------------------------------------------------
1. REGLAS DE ARQUITECTURA (ESTRICTAS)
--------------------------------------------------------------------------------
DeepWiki invalidará el proyecto si no se cumple la separación lógica. Debes programar:

A. CAPA DE PRESENTACIÓN (Presentation Layer):
   - Responsable: Rutas, Controladores y UI.
   - Restricción: PROHIBIDO hacer llamadas a la base de datos o validar lógica de negocio aquí.
   - Acción: Solo recibe la petición, extrae datos y llama a la Capa de Negocio.

B. CAPA DE LÓGICA DE NEGOCIO (Business Logic Layer / Service Layer):
   - Responsable: Validaciones, cálculos de totales, gestión de stock y estados de pedido.
   - Restricción: No conoce nada de HTTP ni de SQL/Consultas directas.
   - Acción: Procesa la información y solicita persistencia a la Capa de Datos.

C. CAPA DE ACCESO A DATOS (Data Access Layer / Persistence):
   - Responsable: CRUD, modelos de datos y manejo de la fuente de datos (Array, JSON o SQLite).
   - Restricción: No contiene lógica de negocio.

--------------------------------------------------------------------------------
2. REQUERIMIENTOS FUNCIONALES
--------------------------------------------------------------------------------
- Catálogo: CRUD de artesanías (Tipo: collares, aretes, manillas; material, precio, stock, artesana).
- Ventas: Registro de pedidos con estados (Pendiente, Enviado, Entregado).
- Promoción: Registro de ferias y productos destacados.

--------------------------------------------------------------------------------
3. TAREAS A REALIZAR POR EL LLM
--------------------------------------------------------------------------------
1. Generar la estructura de carpetas clara:
   /src
     /presentation (controllers/routes)
     /services     (business logic)
     /persistence  (data access/models)
     app.js        (punto de entrada)

2. Implementar 3 componentes base (Artesania, Pedido, Feria).

3. Stack recomendado: Node.js con Express por su claridad para demostrar capas. Usa una base de datos en memoria (Array o JSON) para facilitar la ejecución inmediata en el nodo.

4. Generar un archivo README.md técnico que explique detalladamente cómo se aplicó la arquitectura de 3 capas para que el evaluador de DeepWiki lo lea primero.

5. Proveer un guion de 1 minuto para el video demostrativo.

--------------------------------------------------------------------------------
4. FORMATO DE ENTREGA
--------------------------------------------------------------------------------
Entrega el código completo, archivo por archivo, asegurando que no haya saltos de capas (ej. la capa de presentación nunca debe hablar con la capa de datos directamente).
